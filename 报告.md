# 机器视觉课题 3 —— 跑道形空腔尺寸测量

## 1、概述

本课题要求对工件上的跑道形 (Stadium / Discorectangle) 空腔进行尺寸测量。
被测对象是一个白色塑料 / 金属壳体上的长条形凹腔，中央有矩形凸台 (内亮区)。
共提供 6 张工业相机灰度图 (BMP，2592×1944) 用于算法开发与重复性评估。

需要给出的 4 个测量量：

| 量 | 含义 | 直观比喻 |
|---|---|---|
| r1 | 空腔左端圆弧半径 | 跑道左侧 180° 弧 |
| r2 | 空腔右端圆弧半径 | 跑道右侧 180° 弧 |
| d1 | 跑道形空腔外缘宽度 (外上→外下) | 整个操场的宽度 |
| d2 | 内亮区宽度 (内上→内下) | 草坪 (内场) 的宽度 |

技术约束：

* 仅使用 1D 测量 (测线 + 1D profile + 亚像素边缘检测)
* 亚像素精度
* 兼顾计算时间
* 评估测量重复性

技术栈：**C++17 + OpenCV 4.5**，CMake 构建。
仅使用 OpenCV 的基础数据结构 / I/O / PCA / 形态学 / 旋转矩形，
1D 边缘检测、几何拟合、RANSAC 全部自行实现。

## 3、算法设计

整体流水线：

```
gray → [粗定位空腔位姿] → [径向 1D 扫描 + 圆拟合]    → r1, r2
                       └─ [副轴 1D 扫描 + 4 边识别] → d1, d2
                                                  → 可视化结果图
```

### 3.1 粗定位空腔位姿 (locator.cpp)

目的：得到空腔的几何中心 + 主轴方向 + 长宽尺度，
作为后续 1D 扫描线布置的参考坐标系。该阶段对像素精度要求不高，
真正的亚像素位置完全由后续 1D 边缘检测决定。

步骤：

1. 高斯平滑 → Otsu 二值化 → 形态学开闭：分割出工件 (亮区)。
2. 取最大连通域作为工件掩码 `M_part`。
3. 在 `M_part` 内取灰度低于中位数 `−25` 的像素，得到“空腔候选”二值图，
   再开闭运算 + 取最大连通域，长宽比要求 > 1.8，作为跑道形空腔区域。
4. 对该连通域所有像素做 PCA，主特征向量即主轴方向，
   并把所有点投影到主 / 副轴，分别取 99.5% 分位作为 `length / width`
   (半长 / 半宽，对应跑道两端弧心在主轴坐标系里的偏移 `L − W`)。

核心代码：

```cpp
cv::PCA pca(data, cv::Mat(), cv::PCA::DATA_AS_ROW, 2);
cv::Point2d c (pca.mean.at<double>(0,0),  pca.mean.at<double>(0,1));
cv::Point2d e0(pca.eigenvectors.at<double>(0,0), pca.eigenvectors.at<double>(0,1));
if (e0.x < 0) e0 = -e0;                       // 主轴指向 +x，保证可重复
cv::Point2d e1(-e0.y, e0.x);                  // 右手系副轴
// 投影到主 / 副轴，99.5% 分位作为半长 / 半宽
double half_len = quantileAbs(proj0, 0.995);
double half_wid = quantileAbs(proj1, 0.995);
```

### 3.2 1D 亚像素边缘提取 (measurement.cpp)

这是整个算法的核心精度来源，实现的是工业系统 Halcon `measure_pos`
的等价算法。

**(a) 沿扫描线采样 1D profile**

给定 1D 扫描线 (中心点 `c`，单位方向 `dir`，半长 `L`，半宽 `w`)：

* 主轴方向 `dir` 上，以 1 px 步长采样 `2L+1` 个位置；
* 副轴方向上对称 `±w` 做 `2w+1` 行采样，对每个主轴位置做平均
  (矩形积分)，等价于把一个 `(2L+1) × (2w+1)` 的矩形 ROI 沿主轴方向投影；
* 所有采样均采用双线性插值，保证扫描线可以任意倾斜。

矩形积分的作用：在副轴方向上做相干平均，把空间不相关的噪声 (光子噪声、
读出噪声) 以及局部干扰 (内凸台上的字符标记) 平均掉。本课题选用 `w = 30 px`，
能有效抑制内凸台上间隔约 30 px 的点阵字符。

**(b) 高斯一阶导卷积**

```cpp
// 离散高斯一阶导核 g'(t) = -(t/σ²) · 1/(σ√(2π)) · exp(-t²/(2σ²))
int r = max(1, ceil(3σ));
for (int i = -r; i <= r; ++i)
    g[i+r] = -(i/σ²) · (1/(σ√(2π))) · exp(-i²/(2σ²));
// 归一化：对单位阶跃响应峰值为 1，强度即等价灰度差
```

将 `g'` 与采样 profile 做卷积得到导数信号 `D[t]`。
卷积本质上是 “高斯平滑 + 微分” 的合成算子，
对噪声敏感度按 `1/σ` 衰减，
对尺度小于 σ 的特征 (噪点、字符标记) 严重削弱。

**(c) 极大值 + 抛物线亚像素精化**

对 `|D[t]|` 求局部极大值，过滤掉低于阈值的弱响应；
对每个保留下来的极大值，取 `(t-1, D[t-1])`、`(t, D[t])`、`(t+1, D[t+1])`
三点拟合一条抛物线，顶点位置即亚像素峰位：

$$
t^* = t + \tfrac{1}{2}\,\frac{D[t-1] - D[t+1]}{D[t-1] - 2D[t] + D[t+1]}
$$

只要梯度峰附近 profile 近似抛物线 (这对高斯一阶导响应是严格成立的)，
该公式的理论误差小于 0.05 px。

```cpp
double y1 = D[t-1], y2 = D[t], y3 = D[t+1];
double denom = (y1 - 2.0*y2 + y3);
double dt = 0.5 * (y1 - y3) / denom;        // 抛物线亚像素偏移
EdgePoint p;
p.t        = t + dt;
p.strength = |D[t]|;
p.polarity = (D[t] > 0) ? +1 : -1;
```

边缘极性约定：`+1` 表示沿 `dir` 方向 “暗→亮”，`-1` 表示 “亮→暗”。

**(d) 复杂度**

每条扫描线 O(L·w) + O(L·K)，其中 K=2⌈3σ⌉+1 是卷积核宽度。
本项目 L ≈ 280, w ≈ 60, K ≈ 9，每条扫描线约 2×10⁴ 次乘加，
全图 ~140 条扫描线总约 3×10⁶ 次运算，远小于一次 5MP 图像
拷贝的耗时，因此真正的瓶颈在图像 I/O 与粗定位。

### 3.3 几何拟合 (geometric_fit.cpp)

**圆拟合 (Taubin 1991)**

代数最小二乘的标准算法，对几何分布外推 / 弧度小的弧段也保持
无偏，且不需要迭代代价高的几何最小二乘：

$$
P(x) = A_3 x^3 + A_2 x^2 + A_1 x + A_0 = 0
$$

由四阶矩 `Mxx, Myy, Mxy, Mxz, Myz, Mzz` 构造，
Newton 迭代取最小正根，即可解出圆心和半径。
本项目实测圆拟合 RMS 残差 **0.55 ~ 0.81 px**。

**RANSAC 抗外点**

为防止极少数误识别的弧上点 (例如外形毛刺、灰带瑕疵) 把
Taubin 的解“拖偏”，先用 RANSAC 选三点定圆、统计内点 (内点阈
值 = 1.5 px)、保留最大支持集，再对内点做 Taubin 精拟合。

```cpp
fitCircleRANSAC(pts, /*inlier_thresh=*/1.5, /*iter=*/200,
                center, radius, rms, inlier_idx);
```

### 3.4 测量管线 (pipeline.cpp)

**(a) 测 r1 / r2 —— 径向 1D 扫描 + 圆拟合**

* 左弧心估计：`pose.center − (L−W)·axis_x`
* 在该弧心处以 41 条径向扫描线，扫描角范围 `+95° ~ +265°`
  (覆盖跑道左半圆并避开与直段衔接)
* 每条扫描线只跨越外缘 (空腔灰带 → 工件亮区) 一条 +1 边，
  返回最靠近弧心的 +1 边作为弧上亚像素点
* 对 41 个亚像素点做 RANSAC + Taubin 圆拟合
* 同理 `r2` 用右半圆

```cpp
for (int i = 0; i < N; ++i) {
    double a = a_deg_lo + (a_deg_hi - a_deg_lo) * i / (N-1);
    cv::Point2d dir = cos(a)*cx + sin(a)*cy;
    ScanLine sl{ arc_center + 0.5*(r_inner+r_outer)*dir,
                 dir, 0.5*(r_outer - r_inner), scan_half_width };
    measurePos(gray, sl, σ, threshold, /*polarity=*/+1, peaks);
    pts.push_back(closestToArcCenter(peaks).pos);
}
fitCircleRANSAC(pts, ...);
```

**(b) 测 d1 / d2 —— 副轴 1D 扫描 + 4 边识别**

跑道形空腔垂直方向 (副轴方向) 的灰度典型剖面：

```
255 ─┐                                                     ┌─ 工件
     │outer top                                  outer bot │
~165 │  ┌──── 灰带 ────┐               ┌──── 灰带 ────┐   │
     │  │              │ inner top     │              │   │
     │  │              │ ┌─ 内亮区 ─┐ │              │   │
     │  │              │ │ I≈255   │ │              │   │
255 ─│  │              │ └─────────┘ │              │   │─
     └──┘              └──────────────┘              └────┘
   t = −W              −d2/2     +d2/2             +W
   (axis_y direction)
```

* 在主轴 `axis_x` 方向、跑道直段两端各留 8% 的安全余量，
  均匀布置 `M = 60` 条垂直 (沿 `axis_y`) 扫描线
* 每条扫描线半长 `1.6 W`，半宽 `30 px`，覆盖空腔上下外缘
* 一次 `measurePos` 返回所有亚像素峰，按 t 升序排列
* **关键的 4 边筛选** (`pickFourEdges`)：
  * 外上 = 整条扫描线最外侧 (最小 t) 的 −1 强峰
  * 外下 = 最外侧 (最大 t) 的 +1 强峰
  * 内上 = `(外上, 0)` 区间 t 最大 (最靠中心) 的 +1 峰
  * 内下 = `(0, 外下)` 区间 t 最小 (最靠中心) 的 −1 峰

“最靠近中心” 而不是 “最强” 的内边选择策略有两个动机：

1. 部分零件灰带 → 内亮区之间有亮度渐变带，会同时出现两个 +1 峰
   (渐变起点 + 满亮起点)。我们要测的是“草坪”，所以选满亮起点。
2. 内凸台上的字符标记会产生强而离散的局部峰 (经测幅值可超过
   外缘峰的 1.5 倍)。按 “最靠近中心” 选择，
   恰好跳过这类位于内亮区内部、距中心很近的 ±1 配对峰。

```cpp
double d1 = e_outer_bot.t - e_outer_top.t;   // 跑道整体宽度
double d2 = e_inner_bot.t - e_inner_top.t;   // 内亮区宽度
```

最后对 60 条扫描线的 `d1 / d2` 做 10% 截断均值得到最终测量值，
同时保留每线测值用于评估单图重复性。

* 深紫色双向箭头 + 数值标注：d2，文字落在内亮区中心 (紫线左侧，与 d1 呈左-右对称构图)；

### 4.6 误差源讨论

1. **几何缺陷**：图 3 / 图 6 的外缘存在轻微毛刺与划痕，
   导致圆拟合 RMS 偏大 (0.64 ~ 0.70 px) 但仍在亚像素水平。
   RANSAC (内点阈值 1.5 px) 把这部分异常点剔除后，
   Taubin 圆心 / 半径不受其影响。

2. **字符标记干扰**：内凸台上有点阵打码，单条扫描线最多可
   出现 8 个峰。借助 (a) 主轴方向 30 px 矩形积分 (b) 4 边筛选
   时 "最靠近中心" 的策略，d1 在所有图像的线间标准差仅
   **1.0 ~ 1.6 px**；d2 由于内亮区右端有亮度渐变带，
   不同 x 位置上 4 边检测的取舍略有差异，
   线间标准差 0.5 ~ 15.4 px。后续 10% 截断均值进一步把这部分
   差异压制掉，最终单图 d2 在加噪 30 次时 σ = 0.010 px (CV 0.011%)。

3. **空腔轻微歪斜**：实测主轴 `axis_x` 与图像水平方向夹角 0.5° ~ 1°，
   PCA 自动校正后扫描线沿真实主轴布置，等效消除了倾斜导致的
   测量耦合误差。

4. **r1 / r2 与 d1 的系统差**：实测 `d1 − 2·avg(r1,r2) ≈ 11 ~ 12 px (3.2 ~ 3.6%)`。
   该差源于零件本身：跑道直段中部两侧外缘并非严格相切于端部圆弧，
   而是略向外凸 (注塑脱模圆角或制造公差)，并非算法误差。
   实际上这恰好可以作为零件质量监控的派生量：
   `(d1 − 2·avg_r) / d1` 的散布反映直段相对于圆弧的偏差程度。

## 5、结论

* 完成了课题要求的 4 个尺寸 (r1, r2, d1, d2) 的 1D 亚像素测量，
  并给出工程式可视化结果。
* 跨 6 张图重复性：**CV ≤ 2.13%** (r1=1.11%, r2=1.57%, d1=1.26%, d2=2.13%)，
  主要分量来自真实加工差异。
* 单图加噪重复性 (算法精度本身)：**CV ≤ 0.07%，σ ≤ 0.11 px**，
  其中 d1=0.013 px、d2=0.010 px、r1=0.016 px，全部达到
