## B. 概念问答 (Conceptual Q&A)

### B1. "1D 测量" 是什么意思？为什么用 1D 不用 2D？

**1D 测量** = 沿一条预先确定方向的直线 (扫描线 / 测量线 / measure line) 采样像素，把二维问题降到一维信号处理。Halcon 的 `measure_pos`、Cognex 的 `Caliper Tool` 都属此类。

**用 1D 不用 2D 的理由**：

1. **计算量低**：N 个像素的 1D 卷积是 O(NK)，而二维边缘检测 (如 Canny) 是 O(WHK²)。
2. **抗噪强**：1D 卷积核可设计成"短而锐"的高斯一阶导 (DoG)，无方向不确定性。
3. **方向先验可用**：本项目知道空腔方向，所有边缘都垂直于副轴方向 → 1D 扫描方向天然正确。
4. **亚像素鲁棒**：1D 抛物线拟合只需三个连续像素，几何意义明确；2D 子像素 (如 Förstner / Harris-refine) 误差更大、调参更复杂。
5. **工业兼容**：Halcon / VisionPro 等主流软件就是 1D 流，工程师熟悉。

**本项目的 1D 调用点**：
- 径向扫描线 (r1、r2)：41 × 2 = 82 条
- 副轴扫描线 (d1、d2)：60 条
- 单线长度约 300 ~ 350 px，单条卷积 1 ~ 2 万次乘加，量化后 < 10 µs。

### B2. 什么叫"亚像素 (subpixel) 精度"？

像素 grid 的最小单位是 1 px，但边缘的几何位置可以落在像素之间。**亚像素**就是利用相邻像素灰度推断这个小数位。常见做法：

1. **矩拟合**：用 0~3 阶矩求边缘几何中心
2. **拟合曲线**：对一阶导峰附近 (3~5 点) 拟合二次/高斯曲线，求极值的解析解
3. **重心法**：在阶跃过渡区做一阶矩

本项目用 **方法 2 (三点抛物线)**：
$$
 t^* = t_0 + \frac{1}{2}\cdot\frac{D[t_0-1] - D[t_0+1]}{D[t_0-1] - 2D[t_0] + D[t_0+1]} 
$$
理论上对纯阶跃 + 高斯模糊 (σ ≥ 1) 的边缘，子像素误差 ≤ 0.05 px (Haralick, 1985)。

实测见 §4.3：加噪 30 次后 r1 σ=0.016 px、d1 σ=0.013 px、d2 σ=0.010 px，确实达到亚像素稳定性。

### B3. 为什么用"高斯一阶导 (DoG)"而不是 Sobel / Prewitt？

- **Sobel/Prewitt** 是 3×3 固定核，频谱响应不可调，边缘越锐越好，对噪声敏感。
- **高斯一阶导** $g'(t; \sigma) = -\frac{t}{\sigma^2}\cdot\frac{1}{\sigma\sqrt{2\pi}}\cdot e^{-t^2/(2\sigma^2)}$ 是 **可调带宽** 的最优边缘算子 (Canny 推导)，σ 越大对噪声越鲁棒。本项目 σ = 1.5，核长 9 px。
- 此外 DoG 直接给出 **带符号** 的边缘响应 D[t]，极性 (sign(D)) 即可区分亮→暗 / 暗→亮，无需再分通道。

### B4. 三点抛物线插值的几何意义？

把三点 (t-1, y1), (t, y2), (t+1, y3) 拟合抛物线 $f(t) = a t^2 + b t + c$，对 $f$ 求导取零得峰位 $-b/(2a)$，代数化简后即上面的闭式表达。要点：

- **前提**：极值确为最大；如果三点几乎共线 (denom → 0)，公式退化，需保护 (本项目在 `subpixPeaksFromDerivative()` 里限制 |dt| ≤ 1)。
- **误差源**：当真实边缘 σ 远小于像素 (≪ 1 px) 时，三点窗口外仍有非零响应，抛物线模型欠拟合；可通过对 profile 预先做轻度高斯平滑 (即增大 σ) 来缓解。

### B5. 什么是 Otsu 阈值？为什么本项目用它做粗分割？

Otsu (1979) 在直方图 0~255 上枚举阈值 t，使两类 (前景 / 背景) 的 **类间方差** 最大：
$$ \sigma_b^2(t) = w_0(t)w_1(t)[\mu_0(t) - \mu_1(t)]^2 $$
等价于 **类内方差最小**。复杂度 O(L)，无参数，对双峰直方图最优。

**适用性判断**：工件 (亮) + 背景 (暗) 直方图明显双峰 → Otsu 干脆利落。空腔分割用的不是直接 Otsu，而是 **"工件中位亮度 − 25"** 做相对阈值，因为空腔 (灰) 与工件 (白) 没有真正的双峰，是连续过渡。

### B6. 什么是 PCA？为什么用它求空腔主轴？

PCA = 主成分分析。把空腔像素 (x,y) 看作样本，求协方差矩阵 $C = \frac{1}{n}\sum (p_i - \bar p)(p_i - \bar p)^T$ 的特征向量 / 特征值。最大特征值对应方向 = 散布最大的方向 = 主轴。

- 等价于"对所有像素做最小二乘直线拟合"的 **TLS** 形式。
- 比 `minAreaRect` 抗外点性好：mAR 取的是凸包包围矩形角度，会被毛刺干扰几度，PCA 是全体像素加权平均。
- 本项目两者都算了：PCA 给主轴方向，minAreaRect 留作交叉验证 (保存在 `pose.rrect`)。

### B7. Taubin 圆拟合与 Kasa / Pratt 的区别？

| 方法 | 损失 | 偏置 | 速度 |
|---|---|---|---|
| Kasa 1976 | 代数距离 $\sum (x_i^2+y_i^2 - 2ax_i - 2by_i - c)^2$ | 大半径时严重偏小 | 最快 (线性最小二乘) |
| Pratt 1987 | 同上 + 约束 $a^2+b^2-c=1$ | 中等 | 求解广义特征值 |
| **Taubin 1991** | 加权代数距离 + 雅可比归一化 | **最小** | 求 3 次多项式根，本项目用 Newton 迭代 |
| 几何 (Levenberg) | 几何距离 | 0 (无偏) | 慢，需迭代 + 初值 |

本项目选 Taubin：精度接近几何法，但速度 ~ 10 倍。代码见 `geometric_fit.cpp::fitCircleTaubin`，约 60 行。

### B8. RANSAC 的原理？参数怎么定？

RANSAC = Random Sample Consensus (Fischler & Bolles, 1981)：

```
重复 N 次:
  随机选最少所需点子集 (圆: 3 点)
  解析求模型
  统计内点 (残差 ≤ inlier_thresh)
返回内点最多的模型，用其内点做最终精拟合
```

**本项目参数**：`inlier_thresh = 1.5 px`，`max_iter = 200`。
- **1.5 px** 略大于亚像素噪声 (~0.1 px)，但小于真正几何瑕疵 (毛刺 / 划痕 ~5 px)，能分清"亚像素抖动"和"零件缺陷"。
- **200** 次：3-point 模型理论需要 $N = \log(1-p) / \log(1-w^3)$ 次，p=0.99、w=0.5 时 N=34，留 6 倍冗余足以。

### B9. "矩形积分 (rectangular integration)" 抑制干扰的原理？

扫描线本身只有 1 像素宽，受字符标记 / 划痕影响大。**矩形积分**：在扫描线两侧沿副轴 (即扫描方向的垂线) 各取 ±half_width 像素，**平均** 这些像素的灰度作为 1D profile 的当前值。

数学上等价于对 2D 图像做"沿扫描线方向不动、沿垂线方向平均"的盒子滤波，再切片。
**信噪比 (SNR) 提升 √(2W+1)**：W=30 → √61 ≈ 7.8 倍。

代价：垂直于扫描方向的细结构 (< 2W) 会被洗掉。本项目恰好希望这样：内凸台上的字符宽 ~5 px，远小于积分宽 60 px，被有效抑制。

### B10. 跑道形 (stadium / discorectangle) 几何特征？

跑道形 = 一个矩形 + 两端各一个半圆。参数：
- 中心 (cx, cy)
- 半长 a (主轴方向)
- 半宽 b (副轴方向，同时也是半圆半径)
- 直段长 2(a-b)，圆弧半径 b

本项目 **r1 / r2 ≈ b**，但允许左右独立 (因实际零件不严格对称)；
**d1 ≈ 2b** (跑道宽度)、**d2 ≈ 2b − 2·(灰带厚度)**。

## C. 实现问答 (Implementation Q&A)

### C1. 项目模块怎么划分？为什么？

### C2. 高斯一阶导核怎么离散化的？为什么要归一化？

代码 `measurement.cpp::makeGaussianDerivKernel`：

```cpp
for (int i = -r; i <= r; ++i) {
    double t = (double)i;
    g[i+r] = -(t/s2) * inv * std::exp(-t*t/(2*s2));
}
// 归一化：使核对单位幅度阶跃的响应峰值 = 1
double norm = 0;
for (int i = -r; i <= r; ++i) norm += -g[i+r] * i;
for (auto& v : g) v /= norm;
```

归一化的物理意义：让 **edge_threshold 的数值单位 = 灰度差**。这样 `threshold = 4` 就直接表示"两侧灰度差 ≥ 4 才算边"，与 σ 无关，调参直观。

### C3. 双线性采样为什么够用？为什么不用三次样条 / cubic？

- 双线性 = 4 邻域加权，连续可微 → 抛物线拟合不会被"采样不光滑"破坏。
- 三次样条精度更高但 16 邻域，速度慢 4 倍、还可能 overshoot 在边缘附近反而降精度。
- Halcon `measure_pos` 默认就用双线性，工业标准。

### C4. 怎么从 1D peaks 里识别"外上/内上/内下/外下"四条边？

`pipeline.cpp::pickFourEdges` 的语义筛选：

1. **外上 (-1)**：所有 peaks 中 **t 最小** 的 -1 极性峰 (扫描线最外侧由亮→暗的位置)。
2. **外下 (+1)**：所有 peaks 中 **t 最大** 的 +1 极性峰 (最外侧由暗→亮的位置)。
3. **内上 (+1)**：在 (外上, 0) 区间中，**t 最大** 的 +1 峰 (最靠近中心)。
4. **内下 (-1)**：在 (0, 外下) 区间中，**t 最小** 的 -1 峰 (最靠近中心)。

**关键技巧**："最外侧"识别外缘 → 鲁棒于内部字符标记；"最靠近中心"识别内缘 → 跳过灰带→内亮区的亮度渐变带、直接锁定内凸台真正的边。

### C5. 为什么用 60 条扫描线？多了少了的区别？

- 扫描线数 M 个 → d1 平均值的随机误差 ∝ 1/√M。M=60 时已经把单条 0.1 px 的边缘噪声降到 0.013 px。
- 加倍到 120 条只把 σ 降到 0.009 px，但耗时翻倍 → 边际收益小。
- 单图 M=60 的 60 条副轴扫描线均匀铺在 |x_off| ≤ 0.92·(L-W) 内，避开两端圆弧段 (`width_margin = 0.08`)。

### C6. "10% 截断均值"是什么？为什么不用直接均值或中位数？

把 60 个 d1 测值排序，去掉最大 6 个 + 最小 6 个，对剩下 48 个求均值。
- **中位数 (median)**：完全抗外点，但浪费数据；
- **均值 (mean)**：方差最小，但极易被外点拉偏；
- **截断均值 (trimmed mean)**：折中，抗个别异常 (e.g., 边缘检测偶发误识别) 的同时仍保留 80% 数据的统计强度。

10% 是默认惯例 (Wilcox, 2005)；本项目按零件实际测试，10% 足以拒绝所有偶发异常 (CSV 里看 d2_std 偶尔达 15 px，但 trim 后 d2 仍稳定)。

### C7. 怎么计时？怎么保证测量没受 I/O 影响？

`std::chrono::high_resolution_clock` 在 `measureOne` 入口和出口打点：

```cpp
auto t0 = clk::now();
... 真正算法 ...
out.elapsed_ms = std::chrono::duration<double, std::milli>(clk::now() - t0).count();
```

- **不包含** `cv::imread` 和 `cv::imwrite`，纯算法时间。
- 噪声重复性测试时图像已在内存，无 I/O 抖动 → 70 ms / 张更代表算法真实开销。

### C8. 主程序为什么把噪声重复性单独写成一个块？

- 跨图重复性 (N=6) 把 **算法误差 + 真实加工误差** 揉在一起；
- 同图加噪重复性 (N=30) 让"真实加工误差" = 0，纯粹是算法对噪声的灵敏度；
- 两者对比能定量回答"算法本身有多稳" vs "零件本身有多稳"。

代码见 `main.cpp:149-189`。注入 N(0, 4²) 高斯噪声 (≈ 1.5% 动态范围)，模拟正常工业相机的传感器噪声水平。

## D. 实验问答 (Experimental Q&A)

### D1. 6 张图，d1 的极差是 10 px，是算法不稳吗？

不是。看噪声重复性表 §4.3：同一张图加 30 次噪声测出的 d1 σ=0.013 px。
跨图 4.4 px std 几乎全部来自 **零件真实差异**：6 张图来自不同位置/姿态的实物。

支持证据：跨图 d1 的 std (4.378) ÷ 加噪 d1 的 σ (0.013) = **337 倍**。如果是算法噪声主导，两者应该接近。

### D2. CV 1.1% ~ 2.1% 算好还是坏？

业界经验：
- 视觉测量 CV < 1% → 优秀
- 1% ~ 3% → 良好 (本项目)
- 3% ~ 5% → 可用，需谨慎
- > 5% → 不可信

本项目里 CV 主要分量是零件差异，**算法分量** (噪声重复性 CV) 只有 0.004% ~ 0.064%，远好于行业典型 0.1%。

### D3. 为什么 r1 比 r2 重复性好？

跨图 r1 std=1.86，r2 std=2.62。
原因：右弧端外缘上靠近字符区，**亮度对比稍低** (灰带与字符背景灰度接近)，导致 DoG 响应峰幅值小，受噪声影响更大。表现：r2 加噪 σ=0.109，r1 加噪 σ=0.016，差 7 倍。
若要补救：把右侧径向扫描线的角度范围适当避开字符密集区，或对右弧另设更高的 edge_threshold。

### D4. d2 的 d2_std_px 在图 3 有 15 px，可信吗？

可信。这是 60 条副轴扫描线在 **截断均值前** 的 std，反映的是：内亮区右端的**亮度渐变带宽度** (从灰带到亮内区过渡 30 ~ 50 px) 不同 x 位置不同，pickFourEdges 在不同扫描线上识别到的边略有差异。
但 10% 截断均值后 (`pipeline.cpp:226`)，d2 终值在 6 张图上 std 仅 2.04 px，且加噪重复性 σ=0.010 px → 算法对这种亚结构 **取了一个稳定且语义正确** 的"代表值"。

### D5. 端到端 88.9 ms 里耗时分布怎么样？

按 callgrind 估算 (`-O3 -march=native`，2592×1944)：

| 阶段 | 耗时 (ms) | 占比 |
|---|---:|---:|
| Otsu + 形态学 + 连通域 (locator) | ~52 | 60% |
| PCA (locator 后段) | ~3 | 3% |
| 径向扫描 + 圆拟合 (×2) | ~7 | 8% |
| 副轴扫描 + 4 边筛选 (×60) | ~6 | 7% |
| 可视化 (圆/箭头/文字) | ~15 | 17% |
| 杂项 | ~5 | 5% |

**真正的 1D 亚像素测量 < 15 ms**。如果换更快的粗定位 (e.g., 已知 fixture 跳过 locator)，整条管线可压到 25 ms。

### D6. 如果要把像素转成毫米呢？

需要 **相机标定** (chessboard / dot grid 校准) 得到像素 → mm 的标度 k = mm/px。然后所有输出值乘以 k 即可。本项目把 `imgs/` 里的图像视为已校正图，没有显式的标定步骤，输出统一为像素。报告 §1 已说明。

## E. 进阶 / 挑战题 (Advanced / Stretch Q&A)

### E1. 老师问：为什么不用 Canny + 椭圆拟合？

Canny 是 **2D** 边缘算子，会输出整条边缘链，要再做边链跟踪 + 椭圆 / 圆拟合，流程更长、依赖参数 (高 / 低阈值、连通规则) 多，速度也比 1D 慢一个数量级 (~100 ms vs <10 ms)。最重要的是 Canny 在"内凸台上的字符"附近会产生大量伪边链，反而需要更复杂的后处理来剔除。

### E2. 老师问：为什么不用 Hough 变换检测圆？

Hough 适合**未知位置**的圆 / 直线检测，参数空间投票。本项目空腔粗定位后，圆的位置和近似半径已知 ±5 px，无需投票 → 直接径向扫描 + Taubin 拟合更准、更快 (Hough 量化半径粒度 ≥ 1 px，本身就限制了亚像素精度)。

### E3. 老师问：亚像素精度的理论极限是多少？

Haralick (1985) 推导：对完美阶跃 + 高斯噪声 σ_n，子像素位置的 Cramér-Rao 下界
$$ \sigma_{\text{pos}} \ge \frac{\sigma_n}{\sigma_g \cdot |\Delta I|}\cdot\sqrt{\frac{1}{N}} $$
其中 σ_g 是平滑核宽度，|ΔI| 是阶跃高度，N 是参与平均的像素数。

本项目典型值：σ_n ≈ 4 (灰度)，σ_g = 1.5，|ΔI| ≈ 80 (灰带 vs 工件)，矩形积分 N=61。
代入：σ_pos ≥ 4/(1.5×80)/√61 ≈ **0.004 px**。

实测 d1 σ = 0.013 px，约为理论极限的 3 倍 (因不是纯阶跃，且 RANSAC 引入额外抖动)，已经非常接近。

### E4. 老师问：如果零件大幅倾斜 (e.g., 30°) 会失败吗？

不会。PCA 求主轴对任意旋转鲁棒，扫描线沿真实主轴铺设。但需要注意：
1. 旋转时双线性插值在大角度下精度略降 (~0.01 px)，可考虑切换 cubic；
2. 若旋转 > 45°，需要在 `locator` 中加 **方向归一化** (e.g., 让 `axis_x` 始终指向像素数更多的方向)，否则左右弧的命名可能互换 (本项目已加，见 `locator.cpp:75: if (e0.x < 0) e0 = -e0;`)。

### E5. 老师问：怎么扩展到其它形状 (e.g., 椭圆、矩形)？

模块化设计已经留好接口：
- 椭圆：把 `fitCircle*` 换成 `fitEllipse*` (代数 5-参数最小二乘 + RANSAC)；
- 矩形：径向扫描换成 4 个方向的法向扫描，再用 `fitLine*` 拟合 4 条边；
- 异形：复用 `measurement.cpp` (1D 测量) 不动，只换 `pipeline.cpp` (布线策略)。

`measurement.h` 暴露的 `measurePos()` 就是 Halcon `measure_pos` 的等价接口，本身与形状无关。

### E6. 老师问：单测怎么写？

每个模块都可单独测：
- `measurement.cpp` → 合成"灰度阶跃" profile，验证抛物线峰位误差 < 0.05 px；
- `geometric_fit.cpp` → 生成 100 点圆 + N(0, 0.1) 噪声，验证 Taubin 圆心误差 < 0.01 px、半径误差 < 0.02 px；
- `locator.cpp` → 用合成的跑道形 mask 图，验证 PCA 角度误差 < 0.1°。
- 集成测试 = 本项目主程序 main.cpp，输出 csv 即对照表。

### E7. 老师问：能讲讲你最自豪的一个设计点吗？

**"最靠近中心" 的内边筛选规则** (`pickFourEdges`)。
- 一开始我用"按极性交替"取前 4 个峰，结果被内凸台上字符 (8 个伪峰) 干扰；
- 改成"最强 4 个峰"，又被亮度渐变带 (灰带↔亮内区) 拐成两个 +1 峰中选错的那个；
- 最终发现：**外缘永远是扫描线的极端位置，内缘永远是中心最近的强阶跃**。这两个判据的组合一行代码就把鲁棒性拉到 100%。

代码就 8 行，但找到这个语义需要先理解零件加工导致的亮度分布。这是"算法看似简单、来自实验观察"的典型例子。

### E8. 老师问：项目的局限和未来工作？

1. **粗定位严重依赖 Otsu + 形态学**，遇到光照急剧变化的图像可能失败，可换为 U-Net 语义分割 (代价：模型 + 推理)；
2. **r1 与 r2 当成两个独立圆**，理论上应是 **跑道整体 2D 拟合** (圆 + 直线 + 圆) 而非两个圆，能利用全局一致性进一步降方差；
3. **无相机标定**，输出仅为像素；接入实际产线时需配 1 次 chessboard 标定即可换算 mm；
4. **多核未利用**，60 条副轴扫描线可 OpenMP `#pragma omp parallel for` 并行，估计能再提 3 ~ 4×。

## 附：可能会被问到的"陷阱题"

| 提问 | 简短反应 |
|---|---|
| "你的算法在哪一步用到了机器学习？" | **没有用** ML，全是经典几何 + 信号处理。这是优势：可解释、可调、零训练数据 |
| "如果背景是和工件同色呢？" | locator 会失败。需替换为带光源调度的暗场 / 结构光照明，或用 ML 分割 |
| "你的 σ=1.5 是哪来的？" | 经验值。σ 太小对噪声敏感，太大边缘定位漂移。1 ~ 2 是普遍合理范围 |
| "RANSAC 不重复，每次结果一样吗？" | 我用了固定 seed `mt19937(20260512)`，结果可重复 |
| "你测得的 r1 = 169 px 和真实直径多少 mm？" | 没标定，输出仅像素。若标定 k=0.025 mm/px，则 r1=4.23 mm |
| "为什么不并行？" | 单图 89 ms 已经达标 (>10 FPS)，并行收益不大且会引入抖动；产线节拍如更紧可加 `#pragma omp` |
| "代码注释为什么写中文？" | 项目交付对象是中文使用者；关键算法名 / 数学公式保留英文术语 |
| "你画的圆是亚像素吗？OpenCV `cv::circle` 是整数啊" | 是的，**绘制**是整数 (视觉显示)；**测量** r1 = 169.174 是亚像素，存在 csv / 信息条 |
